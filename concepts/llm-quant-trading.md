# LLM 量化交易

LLM 对传统量化交易范式的增强——不是替代策略，而是解决"何时用哪个策略"和"非结构化信号处理"的核心问题。

## 传统量化策略的致命局限

| 策略类型 | 赚钱场景 | 亏钱场景 | 核心问题 |
|----------|----------|----------|----------|
| CTA 趋势 | 单边大行情 | 震荡/反转 | 不知道何时该停 |
| 网格策略 | 横盘震荡 | 趋势突破 | 不知道何时该撤 |
| 均值回归 | 价格异常偏离 | 基本面突变 | 不知道何为"异常" |
| 技术指标 | 历史规律延续 | 黑天鹅事件 | 滞后性、假信号多 |

## LLM 的核心价值

### 第一层：市场状态识别（策略调度）
- **传统：** 数据 → 信号 → 执行（无上下文）
- **LLM 增强：** 数据 + 新闻 + 政策 + 情绪 + 历史类比 → 市场状态识别 → 策略调度 + 参数动态调整

### 第二层：非结构化信号处理

| 信号类型 | 例子 | LLM 处理方式 |
|----------|------|-------------|
| 政策文本 | 央行报告措辞微调 | 对比历史文本，识别鹰/鸽转向 |
| 社交媒体情绪 | Reddit、KOL 观点 | 提取主题，识别泡沫或恐慌 |
| 产业链情报 | 港口拥堵、工厂开工率 | 结合地理知识推理供应链 |
| 跨市场关联 | 日元贬值→套息平仓→美股波动 | 因果链推理 |
| 历史类比 | 当前是否像 1987 年黑色星期一 | 检索相似情境，生成概率评估 |

### 第三层：元策略生成

人类交易员想法 → LLM 执行流程：
1. 调取相关数据（跨期价差 + 库存报告）
2. 验证相关性是否结构性变化
3. 生成 Python 代码：新策略
4. 自动回测 2019-2026 年表现
5. 输出报告：夏普比率、最大回撤
6. 生成监控 Dashboard

### 第四层：情境化智能风控

**传统硬风控：**
```
IF 回撤 > 5% THEN 清仓
```

**LLM 情境化风控：**
```
当前回撤 4.8%，原因是流动性枯竭而非策略失效，
历史数据显示此类回撤后 72% 概率反弹。

LLM 判断：暂不触发硬止损，但：
  - 降低仓位至 30%
  - 暂停新开仓
  - 生成预警报告给人类监督员
```

## 核心总结

| 核心层面 | 传统量化 | LLM 增强量化 |
|----------|----------|--------------|
| 市场理解 | 仅依赖价格序列 | 价格 + 文本 + 事件 + 情绪 |
| 策略调度 | 固定权重 | 基于市场状态的动态配置 |
| 策略研发 | 人类研究员数月 | 自动化探索+验证+监控 |
| 风险解释 | 单纯数字报警 | 情境化叙事+可解释决策 |
| 市场适应性 | 需定期人工调整 | 实时识别市场制度转换 |
